# Debugging

## Evidence-based debugging

- Follow evidence-first debugging: collect data (logs, traces, tests, debugger output, reproducible steps) before proposing or applying a fix.
- Treat root-cause analysis as hypothesis-first until verified.
- Do not claim causality without proof.

## Communication during debugging

- Avoid overconfidence. Do not present uncertain conclusions as facts.
- State uncertainty explicitly when evidence is incomplete.
- Use evidence-based language: prefer "might", "could", or "one possibility is" before a hypothesis is validated.
- Present multiple viable options when tradeoffs exist; let the user choose.
